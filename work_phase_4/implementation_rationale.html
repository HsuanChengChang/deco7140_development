Implementation Rationale – Key Highlights

This summary highlights the key design and development decisions made for the Food & Culinary Community website in Work Phase 4.

1. Project Overview

The project transforms the A3 Design Proposal into a functional, accessible, and ethical website aligned with the Connect Communities Initiative’s mission to connect people through food.

2. Client & User Alignment

  • Client Goal: Build an inclusive digital hub celebrating cultural diversity through cuisine.
  • User Need: Share recipes, explore events, and engage with local cooks.
  • Result: Responsive layout, reusable components, and an interactive events calendar.

3. Design & Development

  • Applied mobile-first responsive design using CSS Grid and clamp-based typography.
  • Created consistent visuals with style tokens for color, spacing, and shadow.
  • Added interactive events calendar supporting keyboard navigation and .ics export.
  • Implemented a comment feature powered by API, enabling users to post and view feedback dynamically across pages.

4. Accessibility & Ethics

Implemented semantic HTML structure (header, main, nav, footer), ensured keyboard operability, and verified color contrast with IBM Equal Access Checker.

Ethical standards were maintained with inclusive imagery, transparent data consent, and avoidance of dark patterns.

5. Key Changes from Proposal

  • Events Calendar: Added a fully interactive calendar feature to replace the previous static version. The new calendar uses a custom ARIA-compliant grid for improved accessibility and allows users to browse and select event dates.
  • Other Sections: Apart from the calendar addition, most layouts, styles, and interactions remain consistent with the original proposal.

6. Challenges & Solutions

  • Comment Functionality: The main challenge was implementing an interactive comment system directly within the same event page rather than on a separate one. To solve this, I applied JavaScript techniques learned during the Work Phase3, using form submission and dynamic DOM updates to post and display comments without reloading the page.
  • Integration & Testing: Additional effort was required to ensure that API requests and event-specific comment sections worked consistently across different pages.

7. Testing & Deployment

Tested on major browsers and devices. Accessibility reports verified compliance. Deployed on UQCloud and source hosted on GitHub with consistent commits.

8. Future Improvements

  • Community Feature: Currently only a structural framework has been created; interactive functions are planned for future development.
  • Calendar Integration: The calendar can now link directly to event detail pages, and will later support automatic synchronization with new activities.
  • Complete the all the page: including the logo and some small details.

9. Core References

  • WCAG 2.2 – Accessibility Guidelines
  • IBM Equal Access Checker
  • ACS Code of Professional Conduct
  • ARIA Authoring Practices Guide
