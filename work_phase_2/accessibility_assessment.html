Accessibility Assessment — Selected WCAG Success Criteria

Based on our Client Needs (clear, professional, trustworthy content that is easy to find and read) and User Needs (simple recipe steps, optional video, minimal distractions, intuitive navigation, easy and error-proof forms), we selected the following WCAG Success Criteria.

Perceivable

  • 1.1.1 Text Alternatives — Provide alt text for images (recipe photos and step illustrations).
  • 1.3.2 Meaningful Sequence — Ensure recipe instructions follow a logical reading order.
  • 1.4.3 Contrast (Minimum) — Keep sufficient contrast so text is readable.
  • 1.4.4 Resize Text — Support zoom without breaking layout.

Operable

  • 2.1.1 Keyboard Accessible — All navigation and forms operable via keyboard.
  • 2.4.7 Focus Visible — Provide clear focus outline for links and form fields.

Understandable

  • 3.1.5 Reading Level — Use plain language and short sentences.
  • 3.2.3 Consistent Navigation — Keep navigation order consistent across pages.
  • 3.2.4 Consistent Identification — Use consistent labels/icons for repeated elements.
  • 3.3.1 Error Identification — Clearly mark fields with errors in forms.
  • 3.3.2 Labels or Instructions — Provide clear form labels (e.g., “Ingredients”, “Steps”).
  • 3.3.4 Error Prevention — Confirm important submissions and allow corrections.

Not Prioritised (Still Recommended)

  • 1.4.1 Use of Color — Avoid relying on color alone to convey meaning.

Rationale Summary

These criteria directly support clarity, readability, and predictable navigation (Client Needs), while ensuring simple step-by-step instructions, minimal confusion, and safer form submissions (User Needs).
